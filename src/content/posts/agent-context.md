Most agent products treat context as invisible plumbing.

I think context is closer to product surface. It decides what the system can see,
what it can safely ignore, and how much trust a user can place in the next
action.

The better interface is not always the one with more controls. Sometimes it is
the one that makes the active context legible before the model acts.
